# Deploy Turnover Log (Vercel + Render)

| Component | Host | URL pattern |
| --- | --- | --- |
| React client | **Vercel** | `https://your-app.vercel.app` |
| ASP.NET API | **Render** (Docker) | `https://turnover-log-api.onrender.com` |
| Database | **Render PostgreSQL** (free) | via `DATABASE_URL` on API |

---

## 1. API + database on Render

### Option A: Blueprint (recommended)

1. Push this repo to GitHub.
2. [Render Dashboard](https://dashboard.render.com/) â†’ **New** â†’ **Blueprint**.
3. Connect `omniV1/turnover-log` and apply `render.yaml`.
4. After deploy, open the **turnover-log-api** service â†’ **Environment** and set:

   | Variable | Value |
   | --- | --- |
   | `ClientOrigin` | Your Vercel URL, e.g. `https://turnover-log.vercel.app` (comma-separated for multiple) |
   | `Jwt__Key` | Auto-generated by blueprint; keep as-is if already set |
   | `Email__SmtpHost` | **Leave unset** unless you have SMTP (in-app inbox is the default) |

5. Copy the service URL (e.g. `https://turnover-log-api.onrender.com`).
6. Health check: `GET /api/health` should return `healthy`.

**Note:** Free Render web services spin down after inactivity; first request may take ~30s.

### Option B: Manual web service

1. **New** â†’ **PostgreSQL** (free) â†’ note internal connection string.
2. **New** â†’ **Web Service** â†’ connect repo.
3. **Runtime:** Docker  
   **Dockerfile path:** `server/TurnoverLog.Api/Dockerfile`  
   **Docker context:** repository root (`.`)
4. Environment variables:

   ```
   ASPNETCORE_ENVIRONMENT=Production
   DATABASE_URL=<from PostgreSQL dashboard>
   Jwt__Key=<random string, at least 32 characters>
   Jwt__Issuer=TurnoverLog
   Jwt__Audience=TurnoverLogClient
   ClientOrigin=https://your-app.vercel.app
   ```

## 2. Client on Vercel

1. [Vercel Dashboard](https://vercel.com/new) â†’ import `omniV1/turnover-log`.
2. **Root Directory:** `client`
3. Framework preset: **Vite** (or use `client/vercel.json`).
4. **Environment variables** (Production):

   | Name | Value |
   | --- | --- |
   | `VITE_API_URL` | Render API URL, e.g. `https://turnover-log-api.onrender.com` |

5. Deploy.

Preview deployments on `*.vercel.app` are allowed by API CORS automatically. Production URL must be listed in Render `ClientOrigin`.

## 4. Local vs production

| | Local | Production |
| --- | --- | --- |
| Database | SQL Server LocalDB | PostgreSQL (`DATABASE_URL`) |
| Client API | Vite proxy (`VITE_API_URL` empty) | `VITE_API_URL` = Render URL |
| CORS | `http://localhost:5173` | Vercel URL in `ClientOrigin` |

## Troubleshooting

| Symptom | Fix |
| --- | --- |
| CORS error in browser | Set `ClientOrigin` on Render to exact Vercel origin (https, no trailing slash). |
| 401 on all handoffs | Clear site localStorage; log in again. |
| API 502 / timeout | Free tier cold start: wait and retry. |
| DB connection failed | Confirm `DATABASE_URL` is linked to the web service. |
